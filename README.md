# vpn — постоянный туннель в Латвию для macOS (аналог Horizon, только без входов)

Цель: открыл MacBook → весь интернет уже идёт через сервер в Латвии. Надёжно,
незаметно для DPI, со своим зашифрованным DNS, **ничего не нажимая**. Claude,
ChatGPT, YouTube, Telegram и остальной зарубежный интернет работают; российские
сайты и банки остаются на родном IP (быстро и без блокировок «входа из-за рубежа»).

Это **не VPN-приложение, которое надо открывать**, а системный туннель, который
сам поднимается при загрузке и сам восстанавливается после сна — отсюда ощущение
«как корпоративный Horizon, только свой и без логина».

## Как это устроено

```
   MacBook (sing-box TUN, демон launchd)              VPS в Латвии (sing-box)
   ┌───────────────────────────────┐                 ┌──────────────────────┐
   │ весь трафик → туннель          │   443/tcp  ───► │ VLESS+Reality (скрыт)│
   │  кроме .ru/.рф/geoip-ru → напр.│   443/udp  ───► │ Hysteria2 (резерв)   │──► интернет (Латвия/ЕС)
   │ DNS: FakeIP + DoH (без утечек) │                 └──────────────────────┘
   │ авто-failover между протоколами│
   └───────────────────────────────┘
```

- **Сервер** поднимает сразу два протокола: **VLESS + Reality** (TCP/443,
  маскируется под обычный HTTPS к разрешённому сайту — основной, скрытный) и
  **Hysteria2** (UDP/443 — быстрый резерв). Разнородные транспорты редко
  блокируются одной волной одновременно.
- **Клиент** на Mac работает в режиме **TUN** (полноценный VPN на всю систему) и
  через outbound `auto` (`urltest`) сам выбирает рабочий протокол — при волне
  блокировок переключение происходит без участия пользователя.
- **DNS**: FakeIP + DoH (DNS-over-HTTPS) через туннель → провайдер не видит ваши
  домены и не может подменить ответы (DNS-poisoning), нет DNS-leak.
- **Автозапуск**: sing-box запущен как `LaunchDaemon` (`RunAtLoad` + `KeepAlive`).

## Быстрый старт

1. **Сервер.** Заведи VPS в Латвии (root по SSH) и выполни на нём:
   ```bash
   git clone <этот-репозиторий> && cd vpn
   bash scripts/setup-singbox-latvia.sh
   ```
   Скрипт всё установит, сгенерирует ключи и **напечатает значения для клиента**.
   Подробно: [`docs/server-setup-latvia.md`](docs/server-setup-latvia.md).

2. **MacBook.** На Mac выполни:
   ```bash
   bash scripts/install-macos-daemon.sh
   ```
   Введи значения с сервера один раз — дальше всё автоматически.
   Подробно: [`docs/macos-client-setup.md`](docs/macos-client-setup.md).

3. **Проверь:** `https://ipinfo.io` → латвийский IP; `claude.ai` открывается;
   `https://dnsleaktest.com` показывает только наш DoH-резолвер.

## Честная оговорка про «белые списки»

Ни один самостоятельный зарубежный сервер не даёт **100% гарантии** в самом
жёстком режиме белых списков (особенно у мобильных операторов) — это постоянная
«гонка». Но связка **мульти-протокол + авто-failover + Reality на 443 с
правильным SNI** — наиболее устойчивая из реально доступных на 2026 год схем.
Голые OpenVPN/WireGuard в России уже детектируются мгновенно и здесь не
используются. Подробнее — в [`docs/troubleshooting.md`](docs/troubleshooting.md).

## Безопасность секретов

Реальные ключи/пароли/сертификаты **не коммитятся** (см. `.gitignore`). В
репозитории только шаблоны с плейсхолдерами `__ИМЯ__`. Твой рабочий конфиг —
`configs/singbox-client.local.json` (игнорируется git).
