# MacBook: постоянный туннель (sing-box как демон launchd)

Результат: открыл/включил Mac → туннель уже работает. Никаких приложений
открывать и никуда логиниться не нужно. После сна соединение восстанавливается
само.

## Требования
- macOS 13+ (Apple Silicon или Intel).
- [Homebrew](https://brew.sh) (для установки sing-box).
- Значения с сервера (их напечатал `setup-singbox-latvia.sh`).

## Установка «всё сразу» (рекомендуется)

```bash
git clone <этот-репозиторий> vpn && cd vpn
bash scripts/install-all.sh
```
Ставит сразу всё: демон + sudoers без пароля + строку меню + watcher
(failover/health) + алиас `vpn`, и спрашивает про
kill-switch. В конце печатает диагностику. Дальше всё управляется командой `vpn`
или иконкой в строке меню.

Ниже — установка по частям, если нужно вручную.

## Установка демона (по частям)

```bash
bash scripts/install-macos-daemon.sh
```

Скрипт:
1. ставит `sing-box` через Homebrew (если ещё не стоит);
2. спрашивает значения с сервера (или берёт готовый
   `configs/singbox-client.local.json`) и собирает рабочий конфиг;
3. проверяет конфиг (`sing-box check`);
4. кладёт конфиг в `/etc/sing-box/config.json`, а LaunchDaemon — в
   `/Library/LaunchDaemons/com.user.singbox.plist`;
5. загружает демон (`launchctl bootstrap system ...`) и включает автозапуск.

При установке потребуется ввести пароль (`sudo`) — демон ставится системно,
чтобы подниматься ещё до входа в систему.

## Что именно настроено

- **Режим TUN** — весь системный трафик идёт через туннель (а не только браузер).
- **Авто-проверка связи** — outbound `auto` (`urltest`) периодически проверяет
  Reality и держит туннель живым.
- **DNS без утечек** — DoH (DNS-over-HTTPS к `1.1.1.1`) через туннель.
  Провайдер не видит, какие домены вы открываете, и не может подменить DNS.
  Российские домены резолвятся локально и идут напрямую.
- **RU-исключение** — домены `.ru`/`.рф` и российские IP (`geoip-ru`) идут
  напрямую: банки, госуслуги, локальные сайты работают быстро и без блокировок
  «входа из-за рубежа».
- **Чистый TUN без `set_system_proxy`** — специально, чтобы обойти известный баг
  TUN+системный-прокси на свежих macOS.

## Проверка (главное — что работает само)

1. **Статус демона:**
   ```bash
   sudo launchctl print system/com.user.singbox | grep -i state   # running
   ```
2. **Геолокация:** открой <https://ipinfo.io> в Safari → должен быть **латвийский IP**.
3. **Сайты:** `claude.ai`, `chatgpt.com`, YouTube, Telegram — открываются и логинятся.
4. **Нет DNS-leak:** <https://dnsleaktest.com> → виден только наш DoH-резолвер
   (Cloudflare), не российский провайдер.
5. **RU напрямую:** открой российский банк/сайт — он должен работать как обычно
   (идёт мимо туннеля).
6. **Автозапуск:** перезагрузи Mac и, **ничего не нажимая**, повтори п.2 — IP
   снова латвийский.
7. **После сна:** закрой крышку/усыпи, разбуди — туннель восстановился сам.

Логи при проблемах:
```bash
tail -f /var/log/sing-box.err.log
```

## Включение / выключение по желанию

Туннель включается сам при загрузке, но его легко переключать вручную короткой
командой `scripts/vpn.sh`:

```bash
bash scripts/vpn.sh on        # включить
bash scripts/vpn.sh off       # выключить (вернуться к прямому интернету)
bash scripts/vpn.sh restart   # перезапустить (например, после правки конфига)
bash scripts/vpn.sh status    # состояние + текущий внешний IP
```

Чтобы было совсем удобно, добавь алиас в `~/.zshrc` (один раз):
```bash
echo 'alias vpn="bash $HOME/vpn/scripts/vpn.sh"' >> ~/.zshrc && source ~/.zshrc
```
Дальше просто: `vpn on`, `vpn off`, `vpn status`.

> «Выключить» означает выгрузить демон — до следующей перезагрузки туннель
> подниматься не будет. После перезагрузки он снова включится автоматически
> (если только не отключить автозапуск: `sudo launchctl disable system/com.user.singbox`).

### Переключение без ввода пароля (по желанию)
Команды управления демоном требуют `sudo`. Чтобы `vpn on/off` не спрашивал
пароль, добавь правило sudoers (замени `ИМЯ` на свой логин):
```bash
sudo tee /etc/sudoers.d/singbox >/dev/null <<'EOF'
ИМЯ ALL=(root) NOPASSWD: /bin/launchctl bootstrap system /Library/LaunchDaemons/com.user.singbox.plist, \
  /bin/launchctl bootout system /Library/LaunchDaemons/com.user.singbox.plist, \
  /bin/launchctl kickstart *com.user.singbox, /bin/launchctl kickstart -k system/com.user.singbox, \
  /bin/launchctl enable system/com.user.singbox, /bin/launchctl print system/com.user.singbox, \
  /bin/cp /etc/sing-box/config-full.json /etc/sing-box/config.json, \
  /bin/cp /etc/sing-box/config-selective.json /etc/sing-box/config.json, \
  /usr/bin/tee /etc/sing-box/killswitch.pf.conf, \
  /sbin/pfctl -E -f /etc/sing-box/killswitch.pf.conf, \
  /sbin/pfctl -e -f /etc/sing-box/killswitch.pf.conf, \
  /sbin/pfctl -d, \
  /usr/bin/touch /etc/sing-box/killswitch.enabled, \
  /bin/rm -f /etc/sing-box/killswitch.enabled
EOF
sudo chmod 440 /etc/sudoers.d/singbox
```

### Тумблер в строке меню (красиво, в один клик)
В репозитории есть готовый плагин SwiftBar — иконка-индикатор в menu bar с
кнопками включения/выключения и текущей страной выхода.

```bash
bash scripts/install-menubar.sh
```

Что появится в строке меню:
- **🟢 LV** — туннель включён (показывает страну выхода, напр. Латвия);
- **⚪️ off** — выключен (прямой интернет).

Клик по иконке открывает меню: **Включить / Выключить / Перезапустить**, плюс
быстрый доступ к логам. Индикатор сам обновляется каждые 30 секунд и сразу после
любого действия.

> Чтобы кнопки срабатывали без запроса пароля, настрой sudoers (раздел
> «Переключение без ввода пароля» выше) — клик в меню не умеет вводить пароль.

Альтернатива — GUI-клиент **Hiddify** (см. ниже): тоже включение/выключение
кликом, но поднимается только после входа в систему.

## Режимы маршрутизации: весь трафик ↔ только сервисы

Доступно два режима (различаются одним полем в конфиге, переключаются мгновенно):

- **Весь трафик** (`full`, по умолчанию) — через Латвию идёт всё, кроме
  российских ресурсов.
- **Только сервисы** (`selective`) — через Латвию идут только **Claude, ChatGPT,
  YouTube и Telegram**, а весь остальной интернет — напрямую (быстрее, экономит
  канал сервера, меньше «отсвечивает»).

Переключение командой:
```bash
bash scripts/vpn-mode.sh selective   # только сервисы
bash scripts/vpn-mode.sh full        # весь трафик
bash scripts/vpn-mode.sh toggle      # переключить туда-обратно
bash scripts/vpn-mode.sh status      # текущий режим
```

Или прямо из **строки меню**: в меню плагина SwiftBar есть пункт
«→ Переключить на …». В режиме «только сервисы» рядом с флагом страны
показывается значок 🎯.

> Какие именно сервисы туннелируются, задаётся списком доменов в
> `configs/singbox-client.template.json` (секция `route.rules`, правило с
> `outbound: "proxy"`) + rule-set `geoip-telegram` для приложения Telegram.
> Можно добавить свои домены и переустановить демон.

> Переустановка демоном (`install-macos-daemon.sh`) сбрасывает режим на «весь
> трафик». Переключение меню/скриптом требует sudoers без пароля (см. выше).

## Выбор протокола (Reality / Авто)

Сейчас в схеме один протокол — **VLESS+Reality**. Режим **Авто** (`urltest`)
просто следит за его связью; зафиксировать Reality явно можно так — **на лету,
без перезапуска и без sudo** (через локальный Clash API sing-box на
`127.0.0.1:9090`):
```bash
bash scripts/vpn-proto.sh auto        # авто-проверка связи (по умолчанию)
bash scripts/vpn-proto.sh reality     # зафиксировать VLESS+Reality (TCP)
bash scripts/vpn-proto.sh status      # что выбрано и что реально активно
```

В **строке меню** есть подменю «Протокол» с галочкой ✓ у текущего и строкой вида
`Протокол: авто → vless-reality · 42 ms` — видно, какой протокол сейчас реально
несёт трафик **и его пинг**. Пункт «🚀 Проверить пинг сейчас» запускает свежий
замер. Выбор сохраняется в `cache.db` и переживает перезапуск/сон.

### Уведомления о здоровье туннеля
Чтобы получать системное уведомление, когда туннель завис и был автоматически
перезапущен:
```bash
bash scripts/install-watcher.sh
```
Это ставит лёгкий LaunchAgent, который раз в ~20 сек делает **health-check**: если
sing-box завис (процесс жив, но трафик не идёт), после нескольких неудачных проб
подряд он автоматически перезапускает демон и шлёт уведомление «sing-box завис —
перезапущен». Плюс поддерживает актуальным список `utun` для kill-switch. Работает
без sudo. Снять: `launchctl bootout gui/$(id -u) ~/Library/LaunchAgents/com.user.singbox-watch.plist`.

> Это работает без пароля, потому что Clash API слушает только локально
> (`127.0.0.1`) и защищён секретом, который установщик генерирует автоматически и
> кладёт в конфиг.

## Kill-switch (защита от утечки реального IP)

Если туннель упал или выключен, обычно трафик пошёл бы напрямую — и реальный IP
«засветился». Kill-switch через штатный фаервол **pf** этого не допускает: наружу
разрешены только вход в туннель, переподключение к твоему серверу и локальная сеть.

```bash
bash scripts/killswitch.sh on      # включить защиту
bash scripts/killswitch.sh off     # выключить
bash scripts/killswitch.sh status  # состояние
```

Или из строки меню: пункт «🛡 Kill-switch: вкл/выкл». Когда туннель выключен, а
kill-switch активен, в меню видно предупреждение, что интернет заблокирован.

Как это работает: pf блокирует весь исходящий трафик, кроме `lo0`, интерфейсов
`utun*` (вход в туннель), соединений к `SERVER_IP:443` и приватных сетей. Список
`utun*` автоматически обновляется при каждом включении/перезапуске туннеля
(`vpn.sh`) и фоновым watcher'ом — поэтому не устаревает.

**Авто-старт при загрузке:** если kill-switch включён, он поднимается сам вместе с
туннелем при загрузке Mac (демон запускается через обёртку
`/etc/sing-box/daemon-run.sh`, которая включает защиту ещё до старта sing-box).
Включил один раз — и больше об этом не думаешь. (Если обновлял репозиторий — один
раз переустанови демон `bash scripts/install-macos-daemon.sh`, чтобы появилась
обёртка.)

> **Аварийное восстановление:** если что-то пошло не так и интернета нет —
> `sudo pfctl -d` полностью выключает pf.
>
> **Captive-portal** (отель/кафе с веб-логином): под kill-switch страница входа
> не откроется. Временно `killswitch off`, залогинься в Wi-Fi, затем снова `on`.

## Применить изменения конфига
Поправь `configs/singbox-client.local.json` и переустанови:
```bash
bash scripts/install-macos-daemon.sh
```

## Альтернатива с GUI (если не нужен системный демон)
Можно вместо демона использовать приложение **Hiddify** (App Store): импортируй
`vless://…` ссылку (её печатает серверный скрипт), включи в настройках
«запускать при входе» и «авто-подключение последнего профиля». Это проще, но
поднимается только после входа пользователя, а не на этапе загрузки системы.
