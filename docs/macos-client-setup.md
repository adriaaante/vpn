# MacBook: постоянный туннель (sing-box как демон launchd)

Результат: открыл/включил Mac → туннель уже работает. Никаких приложений
открывать и никуда логиниться не нужно. После сна соединение восстанавливается
само.

## Требования
- macOS 13+ (Apple Silicon или Intel).
- [Homebrew](https://brew.sh) (для установки sing-box).
- Значения с сервера (их напечатал `setup-singbox-latvia.sh`).

## Установка

```bash
git clone <этот-репозиторий> vpn && cd vpn
bash scripts/install-macos-daemon.sh
```

Скрипт:
1. ставит `sing-box` через Homebrew (если ещё не стоит);
2. спрашивает значения с сервера (или берёт готовый
   `configs/singbox-client.local.json`) и собирает рабочий конфиг;
3. проверяет конфиг (`sing-box check`);
4. кладёт конфиг в `/etc/sing-box/config.json`, а LaunchDaemon — в
   `/Library/LaunchDaemons/com.user.singbox.plist`;
5. загружает демон (`launchctl bootstrap system ...`) и включает автозапуск.

При установке потребуется ввести пароль (`sudo`) — демон ставится системно,
чтобы подниматься ещё до входа в систему.

## Что именно настроено

- **Режим TUN** — весь системный трафик идёт через туннель (а не только браузер).
- **Авто-failover** — outbound `auto` (`urltest`) каждые ~3 минуты проверяет
  Reality и Hysteria2 и сам выбирает рабочий. Если одну схему придавили — Mac
  молча уходит на другую.
- **DNS без утечек** — FakeIP + DoH (`https://1.1.1.1/dns-query`) через туннель.
  Провайдер не видит, какие домены вы открываете, и не может подменить DNS.
- **RU-исключение** — домены `.ru`/`.рф` и российские IP (`geoip-ru`) идут
  напрямую: банки, госуслуги, локальные сайты работают быстро и без блокировок
  «входа из-за рубежа».
- **Чистый TUN без `set_system_proxy`** — специально, чтобы обойти известный баг
  TUN+системный-прокси на свежих macOS.

## Проверка (главное — что работает само)

1. **Статус демона:**
   ```bash
   sudo launchctl print system/com.user.singbox | grep -i state   # running
   ```
2. **Геолокация:** открой <https://ipinfo.io> в Safari → должен быть **латвийский IP**.
3. **Сайты:** `claude.ai`, `chatgpt.com`, YouTube, Telegram — открываются и логинятся.
4. **Нет DNS-leak:** <https://dnsleaktest.com> → виден только наш DoH-резолвер
   (Cloudflare), не российский провайдер.
5. **RU напрямую:** открой российский банк/сайт — он должен работать как обычно
   (идёт мимо туннеля).
6. **Автозапуск:** перезагрузи Mac и, **ничего не нажимая**, повтори п.2 — IP
   снова латвийский.
7. **После сна:** закрой крышку/усыпи, разбуди — туннель восстановился сам.

Логи при проблемах:
```bash
tail -f /var/log/sing-box.err.log
```

## Управление

```bash
# выключить туннель
sudo launchctl bootout system /Library/LaunchDaemons/com.user.singbox.plist
# включить снова
sudo launchctl bootstrap system /Library/LaunchDaemons/com.user.singbox.plist
# применить изменения конфига: правишь configs/singbox-client.local.json и заново:
bash scripts/install-macos-daemon.sh
```

## Альтернатива с GUI (если не нужен системный демон)
Можно вместо демона использовать приложение **Hiddify** (App Store): импортируй
`vless://…` ссылку (её печатает серверный скрипт), включи в настройках
«запускать при входе» и «авто-подключение последнего профиля». Это проще, но
поднимается только после входа пользователя, а не на этапе загрузки системы.
